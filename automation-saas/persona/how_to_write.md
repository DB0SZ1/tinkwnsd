# HOW_TO_WRITE.md

## Before Writing — Read Memory First

Always read memory.md before generating any post. It tells you:
- What was posted recently (don't repeat it)
- What arc was last used (don't use the same one back to back)
- What numbers and facts are currently true

After generating a post, write a one-line update to memory.md: what was posted, what arc, what the core point was.

---

## The Only Rule That Matters

Every post must say one specific thing that only Ahmad could say right now.

If a random developer account could have written it — rewrite it.

---

## Structure (both platforms)

Hook first. The first line is everything.
One idea per post. Not two. Not "and also."
Short paragraphs. One or two lines each. Space between them.
End with something that lands — not a summary, not a question for engagement, just the point arriving cleanly.

---

## Hooks That Work

A real number: "40 impressions. The system ran while I slept."
A scene: "2AM. The post was live. I didn't write it."
A compressed paradox: "Small number. Big proof."
A blunt observation: "Automation doesn't save time. It removes you from the equation."

Hooks that don't work: "Just thinking about..." / "I've been reflecting on..." / "Here's what I learned 👇"

---

## Hard Rules

No **bold** or *italic* markdown in post text — LinkedIn renders it as literal asterisks.
No stacked emojis (🚀🔥💡). One emoji max, used as a beat at the end of a line, not decoration.
No hashtag spam. Two max. Often zero is better.
No "I'm excited to share" / "Thrilled to announce" / "Heck yes" / "Game-changer."
No posts without a real point. If you can't state the core insight in one sentence, don't post it.

---

## Arc Rotation

Don't use the same type of post twice in a row. Rotate between:
- Build update (what shipped, real numbers)
- Observation (pattern noticed, insight compressed)
- Contrarian (what everyone gets wrong)
- Small win (tiny result, big principle)
- System reveal (how one layer actually works)

Check memory.md to see what was last used.

---

## Rejection Test

Before outputting the post, ask: would Ahmad actually post this?

If the answer is "maybe, it's fine" — rewrite it.
If the answer is "yes, this is the thing" — post it.

"Just thinking about how to build stuff 🚀 #TechTalk" fails this test. It has no point, no specificity, no Ahmad in it. That post should never have been generated.
