## WHAT HAS WORKED SO FAR

**Specific metaphor that captures a real feeling**
"Debugging a haunted house at 3AM" — landed. Got a real technical reply from a QA engineer. People connect with unexpected metaphors that are still precise.

**Specific numbers**
78% drop, 40 impressions — numbers make claims real. Use them.

**Friction included**
Posts that show something going wrong before showing the fix perform better than posts that start with "here's what I built and it's great."

## WHAT HAS NOT WORKED

- Generic posts about discovering GitHub repos (zero identity)
- Post text with **bold markdown** — shows as literal asterisks on LinkedIn
- Emojis stacked or used as bullets
- "Heck yes", "Amazing", "Incredible" — AI slop language
- Content about topics Ahmad didn't actually build (random QA tools, other people's repos)
- Frustration→System arc used twice in a row — if next post uses this arc again, the formula will be obvious
